Lever A — Focus

Increase retention duration

A user who churns after the first order contributes zero to CLV. Fixing first-order experience is the fastest way to extend how long a user stays.

Lever B — Focus

Increase order frequency

Quick commerce unit economics only work at 3–4 orders/month per user. A user stuck at 1 total order makes acquisition spend structurally unprofitable.

Lever C — Not Now

Increase order value

Cross-sells and upsells are useful only after the user trusts the platform enough to return. Fix retention first.

03 — Stakeholders

Who is directly affected?

The delivery partner is out of scope they carry what's already packed. The failure happens upstream, before they're involved.

Stakeholder 1

Consumer

Wants the right order, right weight, every time. A single failure destroys trust that takes many good experiences to rebuild. Zero switching cost — Blinkit is one tap away.

↑ Better F1 experience → stays, reorders, forms habit
Stakeholder 2

Swiggy / Instamart

Needs habitual orderers for unit economics to work. Every churned F1 user is acquisition cost with zero LTV return. High refund and ticket volume is a direct cost drain.

↑ Higher accuracy → lower refunds, higher F1→F2, better CLV
04 — Solution

What product should build

Scanning and weight tools already exist at Instamart. The gap isn't tooling it's accountability and incentive design. Fix the feedback loop, fix the behaviour.

Solution 1 · Ops Product

Real-Time Picker Accuracy Scorecard

A live accuracy dashboard for dark store ops leads — error rate per picker, per shift. A live feed, not a weekly report.

01

Track error rate per picker per shift using existing scan data

02

Surface as a live leaderboard visible to ops lead in real time

03

Alert ops lead if a picker's error rate crosses threshold mid-shift

04

Tie picker accuracy score to weekly incentive structure

Solution 2 · Catalog Product

Substitution Consent Flow + Catalog Sync

Catalog mismatches cause silent substitutions. Build a consent layer and tighten inventory sync so users are never surprised.

01

Flag SKUs with >5% stockout frequency in last 7 days as "low confidence"

02

Push in-app notification before dispatch if substitution occurs — user can accept or cancel

03

Track substitution acceptance rate as a catalog health metric

04

Deprioritise consistently unavailable SKUs from top search results
